# Sniper Doctrine — File Index

This directory contains the **authoritative full text** of all Sniper Doctrine rule files.
These files contain ONLY rules, gates, protocols, scoring methodologies, and voting procedures.
**No sensitive portfolio data** (positions, share counts, cost bases, ammo, fund NAVs) is stored here.

## Relationship to `.sniper-plugin/skills/`

The `.sniper-plugin/skills/` files are **compressed summaries and pointers** used as Claude context.
The files in `docs/doctrine/` are the **authoritative full text** — the primary source of truth.

| If you need... | Use... |
|----------------|--------|
| Quick reference in a Claude session | `.sniper-plugin/skills/*.md` |
| Authoritative rule text (PR review, doctrine amendment) | `docs/doctrine/*.md` |
| Portfolio state (positions, ammo, HWM) | `data/portfolio.json` (gitignored, local only) |

---

## Sensitive Data Location

Portfolio state data (positions, share counts, cost bases, ammo, fund NAVs) is NEVER stored in this
directory. It lives in:

- `data/portfolio_config.json` — gitignored, Commander-filled locally
- `data/portfolio.json` — gitignored, generated by `scripts/import_portfolio.py`
- `command_center/private.enc.json` — AES-GCM encrypted, safe to commit

---

## Amendment Process

All doctrine amendments must follow the PE protocol in `doctrine_pe.md`:
1. Identify the amendment scope (Tier A/B/C)
2. Obtain required council votes
3. Run the 5-step merge procedure
4. Increment the version in this README and in the amended file
5. Commit with message: `doctrine: amend [file] v[old] → v[new] — [brief description]`
