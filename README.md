# 🎯 SNIPER OS

[![CI](https://github.com/carnat/sniper-app/actions/workflows/ci.yml/badge.svg)](https://github.com/carnat/sniper-app/actions/workflows/ci.yml)

Portfolio OS for the **Sniper Doctrine** rules-based investment system. Two tools in one repo:

| Tool | Purpose | How to run |
|------|---------|------------|
| **Streamlit App** (`streamlit_app.py`) | Full portfolio management — transactions, analytics, tax lots, options, news alerts | `streamlit run streamlit_app.py` |
| **Command Center** (`command_center/index.html`) | Static offline dashboard — daily gate check, War Room, backtester | Open in browser or via GitHub Pages |

## Stack

- **Backend:** Python 3.11 · Streamlit · yfinance · pandas · SQLite · ratelimit · cryptography
- **Frontend:** HTML + vanilla JS (no framework) · Chart.js · Canvas 2D · JetBrains Mono + Rajdhani fonts
- **CI/CD:** GitHub Actions — CI tests, daily price pipeline, GitHub Pages deploy

## Quick Start

### 1) Install

```bash
pip install -r requirements.txt
```

### 2) Configure portfolio

```bash
# Generate blank portfolio config template
python scripts/import_portfolio.py --template

# Edit data/portfolio_config.json with your positions (gitignored — never committed)

# Import and validate against Sniper Doctrine rules
python scripts/import_portfolio.py

# Optional: import + encrypt output
python scripts/import_portfolio.py --encrypt
```

See [Doctrine Importing](#doctrine-importing) below for the full schema and validation rules.

### 3) Configure `.streamlit/secrets.toml`

Minimum required structure for the Streamlit app:

```toml
[us_portfolio]
Ticker = ["AAPL"]
Shares = [1.0]
Avg_Cost = [150.0]

[thai_stocks]
Ticker = ["ADVANC.BK"]
Shares = [1.0]
Avg_Cost = [200.0]

[[vault_portfolio]]
Code = "SCBNDQ(E)"
Units = 100.0
Cost = 10.0
Master = "QQQ"
```

News Watchtower setup: [NEWS_SETUP.md](NEWS_SETUP.md)

### 4) Run

```bash
# Streamlit app (server-required)
streamlit run streamlit_app.py

# Command Center (no server needed — open in browser)
open command_center/index.html
```

## Command Center

The Command Center is a static HTML dashboard at `command_center/index.html`. No server required — works offline after the price pipeline has run once.

**Tabs:**
- **Arsenal** — Core + Satellite positions, P&L, thesis/neglect tracking, DCA Matrix
- **Gate Status** — VIX regime, gate logic, standing orders, mandatory sessions
- **Watchtower** — Core (12) and Satellite (3) tickers with tiers, news badges, ADV data
- **Backtester** — DCA simulation with Matrix allocation vs equal-weight comparison
- **War Room** — Canvas 2D pixel art engine with 45 agents (10 Arsenal + 15 Watchtower + 1 VIX Sentinel + 1 Commander + 16 Council + Bear veto), 4 Triad audit functions, consensus detection, regime atmosphere effects

**Data sources:**
- `data/prices.json` — generated by GitHub Actions daily at 09:00 Bangkok (weekdays only)
- `command_center/private.json` or `private.enc.json` — local portfolio data (PIN-encrypted)
- `data/doctrine_state.json` — live thesis/tripwire/blackout state (gitignored)

## Doctrine Importing

The Sniper system uses two data pipelines for doctrine state — **portfolio import** and **live doctrine state**. Both use gitignored JSON files that the Commander maintains locally.

### Portfolio Import (`scripts/import_portfolio.py`)

Imports portfolio positions from a local config file, validates against Sniper Doctrine rules, and outputs a structured JSON consumed by the Command Center and Streamlit app.

**Files:**

| File | Gitignored | Purpose |
|------|------------|---------|
| `data/portfolio_config.json` | ✅ Yes | Commander fills this locally — source of truth for positions |
| `data/portfolio.json` | ✅ Yes | Generated output consumed by the apps |
| `data/portfolio.enc.json` | ✅ Yes | Optional AES-256-GCM encrypted version |

**Workflow:**

```bash
# Step 1: Generate blank template
python scripts/import_portfolio.py --template

# Step 2: Edit data/portfolio_config.json with actual positions
# Step 3: Import and validate
python scripts/import_portfolio.py

# Step 4 (optional): Encrypt
python scripts/import_portfolio.py --encrypt
```

**Config schema (`data/portfolio_config.json`):**

```json
{
  "meta": {
    "version": "1.0.0",
    "last_updated": "2026-04-01",
    "fx_rate": 32.68,
    "hwm_thb": 800000,
    "ammo_stack": [],
    "tier1_cash_thb": 50000
  },
  "arsenal": [
    {
      "ticker": "VRT",
      "shares": 10,
      "avg_cost_usd": 85.50,
      "tier": "Core S2",
      "target_weight_pct": 8,
      "thesis_date": "2026-03-16",
      "thesis_status": "YELLOW_WATCH",
      "flags": ["F3_ZERO"]
    }
  ],
  "vault": {
    "e_class": [],
    "locked": [],
    "krungsri": []
  },
  "shield": [],
  "watchtower": {
    "core": [],
    "satellite": []
  },
  "standing_orders": [],
  "blacklist": []
}
```

**Validation rules (checked at import time):**

| Rule | Code | What it checks |
|------|------|----------------|
| Alpha Filter | Rule 1 | Arsenal/Watchtower tickers not in blacklist (China/banned stocks) |
| Volume Gate | Rule 4 | Warns on 0-share positions — ADV gate must be verified at execution |
| Concentration | DC-40 | Satellite positions capped at 15% of liquid portfolio; Core trim trigger at 40% |
| Drawdown Freeze | DC-59 | Estimated liquid vs HWM — blocks all DCA if below 80% (Freeze), limits to 50% if below 90% (Amber Zone) |

Validation **errors** block the import. Validation **warnings** are displayed but do not block.

### Live Doctrine State (`data/doctrine_state.json`)

A separate gitignored file for real-time thesis status, tripwires, blackouts, and geopolitical watches. The Command Center loads this on startup and injects it into Arsenal arrays and War Room agents.

**Template:** `data/doctrine_state.example.json`

```json
    "ops_version": "1.15.1",
    "council_version": "1.11.0",
    "last_updated": "2026-04-04T18:00:00Z",
    "updated_by": "Commander"
  },
  "theses": {
    "VRT": {
      "date": "2026-03-16",
      "status": "YELLOW_WATCH",
      "note": "F3=0 · 50pts",
      "day0": true,
      "flags": ["F3_ZERO"]
    }
  },
  "tripwires": [
    { "ticker": "APH", "code": "TW-5", "note": "China 15.6% watch", "active": true }
  ],
  "blackouts": [
    { "ticker": "BWXT", "earnings_date": "2026-04-14", "pre_days": 30, "post_days": 7 }
  ],
  "geopolitical": [
    { "ticker": "APH", "watch": "CHINA", "exposure_pct": 15.6, "rule": "DC-46" }
  ],
  "standing_orders": [
    { "id": "SO-3", "desc": "ASTS fill order paused during VIX ORANGE+", "active": true }
  ],
  "regime_overrides": {
    "forced_regime": null,
    "bear_restructure_active": false,

  },
  "council": {
    "active_seats": ["ELDER", "SA", "CA", "ROBOT", "MACRO", "CLOCK", "RISK", "ARCH", "PSY", "IENG", "IO", "GEO", "DSA"],
    "rotating_assignments": {},
    "pending_motions": [],
    "consensus_log": []
  }
}
```

**How to use:**

1. Copy `data/doctrine_state.example.json` → `data/doctrine_state.json`
2. Edit with current thesis dates, tripwires, blackout windows, geopolitical watches
3. The Command Center loads it automatically via `fetchDoctrine()` with fallback chain:
   - `./doctrine_state.json` (in command_center/)
   - `../data/doctrine_state.json` (from repo root)
   - Hardcoded defaults (if neither found)

**Thesis fields per ticker:**

| Field | Type | Description |
|-------|------|-------------|
| `date` | string | ISO date of last thesis review (e.g. `"2026-03-16"`) |
| `status` | string | `INTACT`, `YELLOW_WATCH`, `BROKEN` |
| `note` | string | Short context (e.g. `"F3=0 · 50pts"`, `"earnings watch"`) |
| `day0` | boolean | Whether the Day 0 clock is running for neglect tracking |
| `flags` | array | Active flags: `F3_ZERO`, `EARNINGS_WATCH`, `CHINA_WATCH`, etc. |

## Price Pipeline

```
GitHub Actions (weekdays 09:00 BKK / 02:00 UTC)
  → scripts/fetch_prices.py
    → data/prices.json (committed to repo)
      → Command Center reads on load
```

The pipeline fetches prices for all Arsenal (10) + Watchtower (15) tickers plus VIX and THB/USD via yfinance. Enriched data includes SMA-50/200, 52-week high/low, and regime indicators.

Manual trigger: `python scripts/fetch_prices.py` or via GitHub Actions workflow dispatch.

## Security

- `.streamlit/secrets.toml` — gitignored, never committed
- `command_center/private.json` — gitignored, local dev only
- `command_center/private.enc.json` — AES-256-GCM encrypted (PBKDF2-SHA256, 100k iterations), decrypted client-side via PIN
- `data/portfolio_config.json` / `data/portfolio.json` — gitignored
- `data/doctrine_state.json` — gitignored
- Secret scanner: `python scripts/secret_scan.py` (runs in CI)
- **Commander-in-loop constraint:** No auto-trade, no broker API, no execution automation. All alerts are informational only.
